# Lifted Geometric Data Generation

Here, we provide the scripts to generate pre-training data of GeoPT, which is also the core design of GeoPT.

The key idea is to track dynamic particles with geometries as a sticking boundary (i.e., **lifting geometries with dynamics**).

## Usage

1. Download [ShapeNet](https://huggingface.co/datasets/ShapeNet/ShapeNetCore/tree/main)
2. Prepare the environment.

```bash
pip install -r requirements.txt
```

3. Data generation for Car (ID: 02958343), Airplane (ID: 02691156) and Ship (ID: 04530566).

In `GeoPT_PreTraining_Data.py`, we provide a fast implementation based on [FCPW](https://github.com/rohan-sawhney/fcpw), where we also manually set the bounding box for surrounding points.

```bash
python GeoPT_PreTraining_Data.py --category ship --mesh_root shapenet_folder --save_root save_path
python GeoPT_PreTraining_Data.py --category car --mesh_root shapenet_folder --save_root save_path
python GeoPT_PreTraining_Data.py --category airplane --mesh_root shapenet_folder --save_root save_path
```

On average, each geometry with 100 random walks will take less than 20 seconds on 80 CPU cores (**<0.2s per sample**).

4. Of course, you can apply this data generation to **any kind of geometries**, e.g., the general categories in ShapeNet. 

We have tried pre-trained GeoPT on full ShapeNet (with careful data processing and category reweighting). This can provide further improvement, even though it may contain irrelevant categories, such as furniture.

```bash
python GeoPT_PreTraining_Data_General.py --categories 03001627 03636649 # you can add more cateories...
```

5. Pre-training with dynamics-lifted geometry data.

Please refer to Appendix F.3. for details, where a category-balanced reweighting is adopted for the data loader.

## Check Data

Please be careful about the axes swap and the bounding box setting. You can adopt `--visualize` to check. 

Here is a car example generated by the following scripts

```bash
python GeoPT_PreTraining_Data.py --category ship --mesh_root shapenet_folder --save_root save_path --visualize
```
